# Architecture Context

## Key Decisions

- **Markdown-driven framework**: The entire scrum_workflow framework is defined in Markdown files (commands, workflows, agents, templates, skills) rather than code. The framework's "runtime" is Claude Code interpreting these Markdown definitions as skills/commands.
- **Multi-agent orchestration**: Three core agents (architect, developer, QA) collaborate through structured workflows with cross-talk rounds and synthesis steps. Each agent provides its specialized perspective independently before synthesis.
- **Template-based generation**: All output (stories, refinements, reviews, context files) is generated by filling YAML-frontmatter templates from `scrum_workflow/templates/`. This ensures consistent structure and idempotent regeneration.
- **Pipeline architecture**: Commands trigger workflows, which orchestrate agents and use templates. The flow is: command -> workflow -> agent(s) + template(s) -> output file(s).
- **Copy-based distribution**: The framework is distributed via an NPM installer (`create-scrum-workflow`) that copies Markdown files into the target project's `.claude/skills/` directory and `scrum_workflow/` structure.

## Patterns

- **Architecture Style**: Pipeline / Markdown-as-code framework. The framework itself is a structured set of Markdown definitions interpreted by AI coding assistants.
- **Design Patterns**:
  - Command pattern (commands/*.md trigger workflows)
  - Template method pattern (templates/*.md with variable substitution)
  - Agent pattern (agents/*.md define specialized roles)
  - Skill pattern (Claude Code skills as pluggable capabilities)
  - Pipeline pattern (sequential workflow steps with defined inputs/outputs)
- **Data Flow**: User triggers `/scrum-*` command -> Claude Code loads command file -> Command references workflow -> Workflow orchestrates agent(s) and template(s) -> Output written to `_scrum-output/`

## Dependencies

- **Runtime**: Claude Code (Anthropic CLI) as the execution platform
- **Installer**: Node.js (ES modules), commander ^13.0.0, @clack/prompts ^0.9.0, fs-extra ^11.0.0, js-yaml ^4.1.0, picocolors ^1.1.0
- **Testing**: Vitest ^3.0.0, memfs ^4.0.0
- **Platform targets**: claude-code (primary), github-copilot, opencode, windsurf (defined in config.yaml)

## Constraints

- **Platform-agnostic**: Framework must work across multiple AI coding assistant platforms (Claude Code, Copilot, Windsurf, etc.)
- **No external dependency**: The scrum_workflow framework must be fully independent from external tooling
- **Write boundaries**: Each workflow has explicit write boundaries defining what it may and may not modify
- **Token budgets**: Agent interactions are budgeted (coordination: 4000 tokens, sub_agent: 2000 tokens for claude-code)
- **Idempotent generation**: Context and skill files can be regenerated cleanly without leaving stale artifacts
